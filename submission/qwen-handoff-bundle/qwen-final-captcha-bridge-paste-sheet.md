## Project Description

Paste this description only after the repository, video, static demo, deck, and
Qwen proof links pass public checks. Keep the Alibaba deployment sentence
qualified unless live Function Compute deployment proof is captured.

```text
BidDesk Autopilot is a Qwen-powered multi-agent proposal operations prototype for teams that need to turn messy RFPs and customer email threads into compliant proposal packets under deadline pressure.

The system coordinates five specialized agents. The Intake Agent extracts requirements, deadlines, attachments, and missing facts. The Research Agent prepares account and technical assumptions. The Policy Agent checks pricing, legal, delivery, and compliance constraints. The Quote Agent drafts line items, assumptions, and proposal language. The Approval Agent routes risky commitments to human reviewers before anything is sent.

The demo shows a facilities-maintenance RFI becoming an approval-ready proposal packet with policy flags, quote lines, and human approval questions. The public static demo and local CLI use synthetic data so judges can inspect the workflow without credentials or billing access.

BidDesk Autopilot uses Qwen Cloud through an OpenAI-compatible DashScope endpoint for long-context extraction, policy comparison, structured proposal drafting, and approval memo generation. A redacted live Qwen Cloud run is recorded in the public repository, and the generated report includes a Qwen Cloud Live Summary.

For Alibaba Cloud proof, the public repository includes a Function Compute handler that demonstrates the backend deployment path and Alibaba Cloud service/API integration. Do not read this as a claim that a live hosted endpoint is already available unless the final form also includes separately captured live Function Compute deployment evidence.
```
